Lovable supports AdsGateway via Custom MCP connectors.

## Prerequisites

- A [Lovable](https://lovable.dev) account with an active project
- An [AdsGateway account](https://adsgateway.io)
- A connected Meta Ads account in AdsGateway for analytics and tracking workflows

## Connect AdsGateway

<Steps>
  <Step title="Open Connectors">
    In your project chat, click `+`, open **Connectors**, then **Manage connectors**.
  </Step>
  <Step title="Select Custom connector">
    In **Personal connectors**, choose **Custom**.
  </Step>
  <Step title="Add AdsGateway MCP server">
    Use:

    - Server name: `Ads Gateway`
    - Server URL: `https://ads-gateway.onrender.com/mcp`
    - Authentication: OAuth
  </Step>
  <Step title="Authorize">
    Complete AdsGateway login/authorization in browser and return to Lovable.
  </Step>
</Steps>

### Copy/Paste Agent Prompt (MCP Install Help)

Use this in Lovable chat if it can assist with connector setup:

```text
Help me connect Ads Gateway MCP in Lovable with guardrails.

Requirements:
1) Walk me through Connectors -> Manage connectors -> Custom.
2) Use exactly:
   - Name: Ads Gateway
   - URL: https://ads-gateway.onrender.com/mcp
   - Auth: OAuth
3) Do not change or remove existing connectors.
4) Confirm the connector shows as connected.
5) Give me a verification prompt after setup.
```

### Guardrails

- Use exact MCP URL.
- Keep existing connectors unchanged.
- OAuth only; no custom callback configuration.

## Use AdsGateway for Analytics

Use this sequence:

1. Connect AdsGateway MCP in Lovable.
2. Authenticate and connect Meta.
3. Select the ad account you want to analyze.
4. Run analytics/journey prompts.

Prompt categories:

- Account management
  - `List my Meta ad accounts`
  - `Which account am I currently using?`
- Performance insights
  - `Give me an account summary for the last 7 days`
  - `Are there any anomalies in my ad performance?`
- Journey reports
  - `Generate a campaign report for campaign <id> since 2026-02-01 until 2026-02-07`
- Creative analysis
  - `Analyze the creative for ad <id>`

## Use AdsGateway for Tracking (Pixel + CAPI)

Use this sequence:

1. Start: `setup_meta_tracking` with your `site_url`.
2. Complete each `next_action`.
3. Confirm required checkpoints:
   - `confirm_pixel="CONFIRM_PIXEL"`
   - `confirm_event_plan="CONFIRM_EVENT_PLAN"`
   - `confirm_create_site="CREATE_SITE"`
4. Save your one-time `sk_` key.
5. Implement snippets/server forwarding from `meta_tracking_snippets_get`.
6. Send test event using `confirm_send_test_event="SEND_TEST_EVENT"`.
7. Run `meta_tracking_setup_audit` and apply `next_actions`.

<Warning>
  Never put `sk_` in browser code. Keep it in server/edge environment variables only. Keys are shown once.
</Warning>

## Verify Analytics

Run:

```text
List my Meta ad accounts
```

Then:

```text
Give me my account performance summary for the last 7 days
```

## Verify Tracking

Run both checks:

1. Send a test event from setup verify.
2. Audit setup health:

```text
Audit tracking setup for site_id "<site_id>" over the last 24 hours and list next actions
```

## Troubleshooting

- OAuth popup blocked:
  - enable popups, reconnect connector
- Authorization failed:
  - sign in at `adsgateway.io`, reconnect
- Tracking reconnect required:
  - reconnect Meta when token is not tracking-ready
- Pixel permission recovery:
  - use returned `authorization_url`, grant permission, continue setup
