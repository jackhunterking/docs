---
title: "Tracking Keys"
description: "List key metadata and rotate site keys (keys are shown once)."
---

Tracking keys authenticate requests to the Tracking Gateway.

<Warning>
  Raw `sk_...` values are only shown once at creation/rotation time. Store them securely.
</Warning>

## meta_tracking_keys_list

### What It Does

Lists key metadata for a site (never returns raw keys).

### Inputs

| Parameter | Type | Required | Description |
|----------|------|----------|-------------|
| `site_id` | string | Yes | Tracking site id |

### Output (Stable Fields)

- `success`
- `keys[]` (metadata only: key type, last4, created/revoked timestamps)

---

## meta_tracking_keys_rotate

### What It Does

Rotates keys for a site and returns new raw key values once.

### Confirmation Required

- `confirm_rotate="ROTATE_KEYS"`

### Inputs

| Parameter | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `site_id` | string | Yes | - | Tracking site id |
| `revoke_existing` | boolean | No | true | Revoke existing keys |
| `create_secret` | boolean | No | true | Generate a new `site_key` (`sk_...`) |
| `include_publishable` | boolean | No | false | Also generate legacy `pk_` key (optional) |
| `confirm_rotate` | string | Yes | - | Must be `ROTATE_KEYS` |

### Output (Stable Fields)

- `success`
- `site_id`
- `gateway_endpoint`
- `browser_token_endpoint`
- `keys.site_key` (new `sk_...` when `create_secret=true`)
- `keys.warning`
- `legacy_keys.publishable_key` (only if requested)
- `legacy_keys.secret_key` (compat field)
